# Implementation Plan: Fix P2-001, P2-002, P2-003 (Avatar v1 backlog)

## 1. Plan Goal

Устранить 3 замечания уровня P2 из аудита Avatar v1:

- **P2-001**: Потокобезопасность таймера — `OnSuccessTimerElapsed` вызывает `SetProperty` из thread-pool потока.
- **P2-002**: Дублирование `FakeIrisApplicationFacade` (~80 строк) в `ChatViewModelTests` и `AvatarViewModelTests`.
- **P2-003**: Неиспользуемое свойство `Greeting` в `MainWindowViewModel`.

Цель: минимальные, безопасные правки без изменения архитектуры, публичных контрактов и безопасных поведений.

## 2. Inputs and Assumptions

### Inputs

- **Audit:** `docs/audits/2026-05-01-phase-5-5-avatar-v1.audit.md` — 4 P2-замечания (3 в scope, P2-004 out of scope)
- **Source:** `src/Iris.Desktop/ViewModels/AvatarViewModel.cs`, `src/Iris.Desktop/ViewModels/MainWindowViewModel.cs`
- **Tests:** `tests/Iris.IntegrationTests/Desktop/ChatViewModelTests.cs`, `tests/Iris.IntegrationTests/Desktop/AvatarViewModelTests.cs`
- **Rules:** `.opencode/rules/iris-architecture.md`, `.opencode/rules/dotnet.md`, `.opencode/rules/verification.md`
- **Verification evidence:** 125/125 tests pass, 0 build errors

### Assumptions

1. `Avalonia.Threading.Dispatcher.UIThread` доступен в `Iris.Desktop` (транзитивная зависимость от Avalonia).
2. Извлечение `FakeIrisApplicationFacade` в общий `internal` класс в тестовом проекте не нарушает архитектуру (тесты — вне production-assembly).
3. `Greeting` не используется ни в XAML, ни в коде, ни в тестах.
4. Правки не требуют новых NuGet-пакетов или проектных ссылок.

## 3. Scope Control

### In Scope

- P2-001: Обернуть `State = AvatarState.Idle` в `OnSuccessTimerElapsed` в `Dispatcher.UIThread.Post`.
- P2-002: Создать общий `FakeIrisApplicationFacade.cs` и `SendCall` record; удалить дубли из обоих тестовых файлов.
- P2-003: Удалить `Greeting` из `MainWindowViewModel`.

### Out of Scope

- P2-004 (FallbackPanel visibility) — оставляется для Avatar v2.
- Любые изменения в Application/Domain/Persistence/ModelGateway.
- Новые тесты или изменение тестовой логики.
- Изменение XAML `MainWindow.axaml` (кроме косвенного — удаление неиспользуемого binding).

### Forbidden Changes

- Не модифицировать `ChatViewModel`, `AvatarViewModel` (кроме P2-001), `AvatarPanel`, `AvatarOptions`, конвертеры.
- Не добавлять проектные ссылки или пакеты.
- Не менять публичные контракты (`AvatarViewModel`, `MainWindowViewModel` конструкторы, `AvatarOptions`).
- Не запускать destructive команды.
- Не добавлять новые требования или спекулятивные абстракции.

## 4. Implementation Strategy

Три независимых исправления, которые можно выполнять в любом порядке. Для минимизации риска рекомендуется:

1. **P2-003 (Greeting)** — самое простое и безопасное, удаление мёртвого кода.
2. **P2-001 (Timer thread safety)** — однострочный фикс, требует проверки таймер-тестов.
3. **P2-002 (Test dedup)** — рефакторинг тестов, безопасен при сохранении поведения.

Каждая фаза самостоятельна и откатывается отдельно.

## 9. Risk Register

| Risk | Impact | Mitigation |
|---|---|---|
| P2-001 `Dispatcher.UIThread.Post` недоступен в тестах | Medium | `Dispatcher.UIThread` — singleton из Avalonia. В headless-тестах xUnit он может быть `null` или не инициализирован. Проверить: если тесты падают, обернуть в `Dispatcher.UIThread?.Post(...)` с fallback на прямой вызов. |
| P2-002 namespace конфликт после извлечения | Low | Новый файл в `Testing/` namespace. Обеспечить `using Iris.IntegrationTests.Testing;` в тестовых файлах или использовать file-scoped namespace. |
| P2-002 `SendCall` или `FakeIrisApplicationFacade` имеют незначительные различия между файлами | Low | Сравнить перед извлечением. Если есть различия — слить в одну реализацию (обе тестовые копии должны работать с единой реализацией). |
| P2-003 `Greeting` используется в design-time | Low | Проверить `MainWindow.axaml` на binding к `Greeting`. Если используется — удалить binding. |
